# 论文阅读辅助网页项目计划

## 1. 项目背景

在日常读论文时，常见痛点是：
- 不确定一篇论文是否值得深入阅读；
- 花大量时间做初步摘要、翻译和摘录；
- 在决定是否放入 Zotero 前，缺少快速判断依据。

本项目目标是做一个网页工具，支持上传论文（链接或 PDF），自动生成论文概括、关键句注释和中英文对照翻译，帮助用户快速判断论文价值，再决定是否纳入 Zotero 进行系统整理。

## 2. 项目目标

构建一个可本地运行的 Python 网页应用，实现：
- 论文内容导入（URL / PDF）；
- AI 自动概括论文；
- 关键句提取并生成注释；
- 英文到中文翻译，左右分栏中英文对照展示；
- 结果可复制/导出，支持后续人工整理。

## 3. 功能需求（第一版 MVP）

### 3.1 上传与解析
- 支持输入论文链接（优先 arXiv 页面或 PDF 直链）；
- 支持本地 PDF 上传；
- 统一抽取论文文本（标题、摘要、正文片段）。

### 3.2 AI 概括与阅读判断
- 基于大模型 API（如通义千问）生成：
  - 论文一句话总结；
  - 研究问题与核心贡献；
  - 方法要点；
  - 适用场景和局限；
  - “是否值得精读”建议（含理由）。

### 3.3 关键句注释
- 自动抽取若干关键句（例如 5-10 句）；
- 对每句生成简短注释，解释其意义（方法创新点、实验结论、限制条件等）；
- 用户可手动补充或修改注释。

### 3.4 翻译与中英文对照
- 将英文内容翻译为中文；
- 前端使用左右分栏形式展示：
  - 左栏：英文原文；
  - 右栏：中文翻译；
- 支持按段落对齐显示，便于对照阅读。

### 3.5 结果管理（基础）
- 显示并保留本次分析结果；
- 一键复制摘要/注释；
- 预留导出为 Markdown 的能力（便于粘贴进 Zotero 笔记）。

## 4. 技术方案

## 4.1 前端框架选择建议

### 方案 A：Streamlit（推荐）
优点：
- 上手快，适合 Python 端到端原型；
- 组件较全（文件上传、文本输入、分栏布局）；
- 对“左右中英文对照 + 结果面板”实现简单。

缺点：
- 精细化前端交互能力有限；
- 样式定制不如传统前端框架灵活。

### 方案 B：Gradio
优点：
- 快速搭建 AI Demo 很方便；
- 输入输出组件简单直接。

缺点：
- 更偏模型演示场景；
- 复杂页面布局（多区块阅读器）可控性略弱于 Streamlit。

结论：
- 第一版建议用 `Streamlit`，在保证开发效率的同时，足以实现论文阅读辅助页面；
- 后续若要做复杂交互，再考虑迁移到前后端分离架构（FastAPI + Vue/React）。

## 4.2 后端与服务
- 应用框架：Streamlit（单体应用）；
- 文本抽取：
  - PDF：`pypdf` 或 `pdfplumber`；
  - URL：`requests + BeautifulSoup4`（按网站规则抽取）；
- AI 接口：通义千问 API（可通过官方 SDK 或 HTTP 调用）；
- 配置管理：`.env` 存放 API Key；
- 数据存储（MVP）：本地 JSON/Markdown 文件。

## 4.3 模块划分
- `ui/`：页面与交互逻辑；
- `services/parser.py`：链接/PDF 文本解析；
- `services/llm.py`：大模型调用封装；
- `services/pipeline.py`：摘要、注释、翻译流程编排；
- `models/schema.py`：结果数据结构定义；
- `exports/`：结果导出（Markdown）。

## 5. 关键流程设计

1. 用户上传 PDF 或输入链接。  
2. 系统解析并提取正文（必要时分段/分块）。  
3. 调用 LLM 生成概括信息与“是否值得读”建议。  
4. 抽取关键句并生成注释。  
5. 执行翻译，按段落形成中英对照。  
6. 页面展示：摘要卡片 + 关键句注释 + 左右对照阅读区。  
7. 用户复制或导出结果到本地，后续手动整理进 Zotero。

## 6. 非功能需求
- 响应速度：单篇论文初次处理控制在可接受范围（例如 10-60 秒，取决于长度与 API 延迟）；
- 稳定性：接口报错时给出重试与错误提示；
- 安全性：API Key 不写死在代码中；
- 可维护性：模块化设计，便于替换模型和解析器。

## 7. 风险与应对
- PDF 抽取质量不稳定：对抽取结果增加清洗与段落修复；
- 超长论文导致 token 成本高：分块处理 + 分层摘要；
- 翻译与术语准确性问题：提供“重新翻译该段”按钮；
- 不同链接结构差异大：先优先支持 arXiv，再逐步扩展站点解析规则。

## 8. 开发计划（建议）

### 里程碑 1：MVP 基础（1-2 天）
- 完成 Streamlit 页面骨架；
- 支持 PDF 上传和 URL 输入；
- 完成文本抽取和基本展示。

### 里程碑 2：AI 能力接入（1-2 天）
- 接入通义千问 API；
- 实现摘要、关键句注释、段落翻译；
- 实现左右分栏中英文对照。

### 里程碑 3：可用性优化（1 天）
- 加入加载状态、异常提示；
- 优化提示词质量；
- 增加结果复制与 Markdown 导出。

## 9. 验收标准（第一版）
- 能成功处理至少一篇英文论文（链接或 PDF）；
- 页面可展示完整摘要、关键句注释和中英对照翻译；
- 用户可据此判断论文是否值得放入 Zotero；
- 应用可在本地稳定运行并重复使用。

## 10. 后续可扩展方向
- 直接对接 Zotero API 自动建条目与写入笔记；
- 增加“术语表”和“方法流程图”自动生成；
- 增加多模型切换（千问 / DeepSeek / OpenAI）；
- 支持多篇论文对比阅读与横向评估。
